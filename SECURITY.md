# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in [Murmur](https://murmurtts.com/), please report it privately so it can be addressed before public disclosure.

**Email:** [tarunyadav9761@gmail.com](mailto:tarunyadav9761@gmail.com)

Please include:
- A clear description of the issue
- Steps to reproduce
- The Murmur version affected (Help → About in the app)
- Your macOS version

You can expect an initial response within 72 hours.

## Privacy

Murmur runs entirely on your Mac. The app makes no outbound network requests for synthesis, voice cloning, or telemetry. Your text, audio, and cloned voices stay on your device.

For full details, see the [Murmur privacy policy](https://murmurtts.com/privacy).

## Update integrity

Murmur uses [Sparkle](https://sparkle-project.org/) for in-app updates with cryptographic signature verification. Update keys were rotated in version 1.0.5 (April 2026) for stronger update integrity. See [CHANGELOG.md](./CHANGELOG.md) for details.

## Supported versions

Only the latest released version of Murmur receives security updates. [Download the latest build →](https://murmurtts.com/)
